- Producing outputs that integrate directly into project folder structures

For the InvenTree API automation, Cursor is ideal because it can:
- Convert structured manual test cases (generated by Claude) into ready-to-run scripts

## Purpose:

The purpose of this prompt is to:
- Generate automation scripts for InvenTree API based on manual test cases
- Automate testing of all key endpoints (Parts, Part Categories, etc.)
- Validate:
    - CRUD operations
    - Filtering, search, pagination
    - Field-level validations and error handling
    - Relationships between entities
    - Authorization and authentication
    - Edge and negative scenarios
- Structure the automation framework with supporting files for reusability and maintainability

## Prompt Usage:

The automation scripts generated by Cursor should follow these guidelines:

### 1. Folder Structure
      automation/api/tests/ → all API test scripts
      automation/api/conftest.py → setup, teardown, and fixtures
      automation/api/config.py → base URL, environment, authentication
      automation/api/requirements.txt → Python dependencies

### 2. Test Script Requirements
Each script should:
- Be modular and reusable
- Include request URL, method, headers, payload, and expected response
- Contain assertions for response status, schema validation, and business rules
- Include comments for clarity
- Include both positive and negative test cases derived from Claude’s manual test cases

### 3. Test Coverage
- CRUD operations for all entities
- Filtering, search, pagination functionality
- Field-level validation rules and boundary conditions
- Relational integrity between entities
- Invalid payloads and edge cases

### 4. Output Requirements
- Generate scripts in Python using pytest + requests
- Scripts should be ready to run in CI/CD pipelines
- Maintain consistent code style, modularity, and clear naming conventions

## Prompt:

You are a Senior QA Automation Engineer.

Generate API automation scripts for the InvenTree Parts API using Python.

The scripts must be derived from the manual API test cases already generated in the project as the source of truth.

Manual API test cases are located in:

test-cases/api-manual-tests/

Files to reference:

test-cases/api-manual-tests/part_crud_tests.md
test-cases/api-manual-tests/category_crud_tests.md
test-cases/api-manual-tests/filtering_pagination_tests.md
test-cases/api-manual-tests/field_validation_tests.md
test-cases/api-manual-tests/relational_integrity_tests.md

Convert these manual test cases into executable API automation scripts.

Automation Framework Requirements:

Language: Python
Test framework: pytest
HTTP client: requests

Execution environment:

Assume a running InvenTree instance via Docker.

Base URL:
http://localhost:8000/api

Automation scripts must satisfy the following requirements:

1. Scripts must be executable against a running InvenTree instance.
   Assume the API is running locally via Docker.

   Base URL:
   http://localhost:8000/api

2. Cover the following scenario types:
   - Positive scenarios
   - Negative scenarios
   - Boundary scenarios

3. Include assertions for:
   - HTTP status codes
   - Response schema / response fields
   - Business logic validations (e.g., correct category assignment, correct filtering results)

4. Demonstrate data-driven or parameterized testing where applicable.
   Use pytest parameterization for cases such as:
   - filtering
   - pagination
   - validation scenarios

5. Maintain traceability with the manual test cases by referencing the Test Case ID in test names or comments.

6. Use good automation practices:
   - reusable configuration
   - pytest fixtures
   - clean and readable test functions
   - descriptive test names

Project structure for generated files must be:

automation/api/

automation/api/config.py
automation/api/conftest.py
automation/api/requirements.txt

automation/api/tests/

automation/api/tests/test_parts_api.py
automation/api/tests/test_categories_api.py
automation/api/tests/test_filtering_pagination.py
automation/api/tests/test_validation.py

Implementation guidelines:

config.py
- store the base API URL
- store reusable configuration variables

conftest.py
- create pytest fixtures such as base_url and headers
- include any reusable setup utilities

requirements.txt
- include required dependencies for running the tests

Test files inside automation/api/tests should:
- implement API calls using requests
- include assertions for status codes and response data
- reference the manual test case IDs
- include parameterized tests where appropriate

For each generated file, clearly label the file name so the code can be copied directly into the project folder structure.
